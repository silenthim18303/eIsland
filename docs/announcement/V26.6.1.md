> **Release Date:** *`2026-06-12`* **GitHub Repository:** *`https://github.com/JNTMTMTM/eIsland`*

**V26.6.1 新增相册媒体分组与多选管理、ESA CDN 更新源与 MinIO 上传支持，同时强化了存储安全校验与媒体协议访问限制，并修复了多项影响稳定性的问题。**

*V26.6.1 introduces album media grouping and multi-select management, ESA CDN update source and MinIO upload support, along with enhanced storage security validation and media protocol access restrictions, plus multiple stability fixes.*

## 新功能

- 新增相册媒体分组功能，支持按文件夹或日期对媒体进行归类整理。
- 新增相册媒体多选功能，支持点选切换与批量操作。
- 新增 ESA CDN 更新分发源，提升更新下载速度与可用性。
- 新增 MinIO 上传集成，替代原有宝塔面板上传方案。
- 新增公告内容 HTML 净化处理，防范潜在的 XSS 安全风险。
- 限制媒体协议仅可访问用户数据壁纸目录，防止越权读取文件。

## 体验优化

- 优化相册网格悬停效果，新增缩放变换动画，交互反馈更生动。
- 优化相册媒体选中状态样式，勾选位置与间距更合理。
- 优化翻译工具下拉菜单的外部点击关闭逻辑，操作更可靠。
- 优化同一曲目播放状态更新时的时间线数据保留，进度显示更稳定。

## 问题修复

- 修复壁纸文件路径未规范化导致读取失败的问题。
- 修复存储操作缺少键名校验可能导致数据异常的问题。
- 修复相册选中勾号定位偏移与多余内边距的问题。
- 修复测试输出不便于排查的问题，测试日志现统一输出至文件。
- 修复 CI 工作流中测试步骤因退出码处理不当而误报失败的问题。
- 修复 CI 环境中缺少 Electron 二进制文件导致测试无法运行的问题。

## 文档更新

- 更新 Node.js 类型声明版本与 TypeScript 配置。
- 补齐壁纸捕获测试路径以匹配新的目录结构。

感谢大家持续反馈与支持。若你在升级后发现新问题，欢迎继续反馈，我们会尽快跟进。

## New Features

- Added media grouping in the album view, supporting classification by folder or date.
- Added multi-select for album media, supporting toggle selection and batch operations.
- Added ESA CDN as a new update distribution source for faster and more reliable downloads.
- Added MinIO upload integration, replacing the previous BaoTa panel upload method.
- Added HTML sanitization for announcement content to prevent potential XSS security risks.
- Restricted the media protocol to only access the user data wallpapers directory, preventing unauthorized file access.

## Improvements

- Optimized album grid hover effects with new scale transformation animations for more vivid interaction feedback.
- Optimized album media selection styles with improved checkmark positioning and spacing.
- Optimized click-outside dismissal logic for the translation tool dropdown menu for more reliable operation.
- Optimized timeline data retention during playback-only updates of the same track for more stable progress display.

## Bug Fixes

- Fixed wallpaper file path normalization issues that caused read failures.
- Fixed missing store key validation that could lead to data corruption.
- Fixed album selection checkmark positioning offset and unnecessary padding.
- Fixed test output to redirect logs to files for easier debugging.
- Fixed CI workflow test steps incorrectly reporting failures due to improper exit code handling.
- Fixed CI environment missing Electron binary that prevented tests from running.

## Documentation

- Updated @types/node version and TypeScript configuration includes.
- Updated wallpaper capture test paths to match the new directory structure.

Thank you for your continued feedback and support. If you encounter new issues after upgrading, please continue to report them — we will follow up as soon as possible.
